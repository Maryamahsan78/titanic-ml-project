# Titanic Survival Predictor

A machine learning project that predicts whether a Titanic passenger would
have survived, based on their details — trained with scikit-learn and
served through an interactive Streamlit web app.

## What's in this project

```
titanic-ml-project/
├── data/
│   └── train.csv          # Titanic dataset (891 passengers)
├── train_model.py         # Trains + compares 3 models, saves the best one
├── app.py                 # Streamlit web app for live predictions
├── requirements.txt       # Python dependencies
├── titanic_model.pkl      # Saved trained model (generated by train_model.py)
├── scaler.pkl             # Saved feature scaler (generated)
├── encoders.pkl           # Saved label encoders + metadata (generated)
└── feature_importance.csv # Feature importance scores (generated)
```

## What the model does

- Engineers features from raw data: extracts **Title** (Mr/Mrs/Miss/Master/Rare)
  from passenger names, computes **FamilySize** and **IsAlone**, and flags
  **HasCabin** as a proxy for missing cabin records.
- Fills missing Age using the median *per title group* (more accurate than a
  flat median).
- Trains and compares three models: Logistic Regression, Decision Tree, and
  Random Forest — then automatically keeps whichever scores highest.
- On this dataset, **Random Forest wins at ~81.6% test accuracy**.

## Local setup

### 1. Install Python dependencies

```bash
cd titanic-ml-project
pip install -r requirements.txt
```

(If you're on a system that requires it: `pip install -r requirements.txt --break-system-packages`)

### 2. Train the model

```bash
python train_model.py
```

This reads `data/train.csv`, engineers features, trains all three models,
prints accuracy/confusion matrix/classification report for each, and saves
`titanic_model.pkl`, `scaler.pkl`, and `encoders.pkl`. You only need to run
this once (or again if you change the feature engineering).

### 3. Run the app locally

```bash
streamlit run app.py
```

This opens the app in your browser at `http://localhost:8501`. Fill in
passenger details and hit **Predict survival** to see the live result.

## Deploying it for free (so you can share a public link)

1. Create a new GitHub repo and push this whole folder to it (including the
   `.pkl` files generated by `train_model.py` — commit those too, so the app
   doesn't need to retrain on deploy).
2. Go to **[share.streamlit.io](https://share.streamlit.io)** and sign in
   with GitHub.
3. Click **"New app"**, select your repo, branch, and set the main file path
   to `app.py`.
4. Click **Deploy**. Streamlit Cloud installs `requirements.txt` and gives
   you a public URL like `yourname-titanic-app.streamlit.app`.

That URL is what you put on your CV/LinkedIn — a live, clickable demo is far
more convincing to a recruiter than a link to a Jupyter notebook.

## Ideas to extend it (good for your CV write-up)

- Add a SHAP or permutation-importance explanation of *why* the model made
  a specific prediction, not just what it predicted.
- Add cross-validation instead of a single train/test split, and report
  the mean ± std accuracy.
- Try XGBoost or LightGBM and compare against the current best model.
- Add a "compare passengers" mode so users can see two predictions side by
  side (e.g. same passenger, different class).
